Viri
Cuando se trabaja en diferentes branchs la union de ellas puede ocasionar conflictos, esto sucede cuando los commits se quieren fusionar y la rama actual modifica la misma parte de un archivo en particular, git no puede decidir cual version guardar y es cuando suceden los conflictos.

Estefanía:
Al trabajar en el mismo archivo surgen diversos cambios que pueden afectar a las mismas lineas de código
lo cual genera conflictos, los cuales deberán ser resueltos por el desarrollador que realice el 
merge. Existirá la opción de "abortar" el merge, aceptar ambos cambios, elegir entre uno de los cambios,
o incluso fusionar ambos cambios. Dichos conflictos pueden ser solucionados en la terminal o desde el
editor de código.

David: Conflicts occurs when two developers modified the same line or when a developer
delete a file while the other is working on it.

Luis: Los conflictos se dan cuando se trata de hacer merge y se encuentran archivos
que fueron modificados mas de una vez en la misma linea. Para resolverlos
hay que decidir que cambios se quedan o como combinarlos.

Ricardo: A veces hay conflictos a la hora de editar un archivo cuando mas de alguna persona trabaja sobre el mismo y modifica las mismas lineas.
Solucionar estos conflictos no es una cosa muy dificil pero puede llegar a complicar un poco las cosas a la hora de colaborar.

Emmanuel: Git no une dos archivos automaticamente si hay modificaciones en ambos en la misma linea(s),
es necesario resolver los conflictos manualmente y hacer un nuevo commit con las cambios necesarios para que funcione o simplemente abortar la union y los archivos vuelven al estado en el que estaba antes de iniciar el merge.

Hector: En ocasiones, cuando dos o mas desarrolladores trabajan en un mismo archivo, pueden surgir conflictos
debido a que se trabajaron en las mismas lineas de texto del archivo. 
Cuando esto sucede, Git no tiene conocimiento de como resolver tal situacion, puesto que no sabe cual version 
o cambio del archivo utilizar para los repositorios.
En estos casos, es tarea del desarrollador que esta haciendo merge de otra branch a la suya el resolver los conflictos
de forma manual. Afortunadamente muchos editores de texto y IDEs ofrecen soluciones sencillas para la resolucion de estos.
